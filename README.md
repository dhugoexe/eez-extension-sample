# eez-ext-sample

A sample EEZ Studio extension demonstrating how to create action components, custom struct/enum types, and object variable types using the EEZ Flow extension API.

## Overview

This extension registers the following in EEZ Studio:

- **2 action components** — `GetData` and `SetData`
- **2 struct types** — `channel_config` and `device_data`
- **1 enum type** — `ChannelStatus`
- **1 object variable type** — `Data` (with a built-in configuration dialog)

## Project Structure

```
eez-ext-sample/
├── src/
│   ├── index.ts                  # Extension entry point, registers all types
│   ├── flow-components.ts        # Action component definitions
│   ├── object-variable-types.ts  # Object variable type with dialog editor
│   ├── eez-types.ts              # EEZ Studio TypeScript type definitions
│   └── tsconfig.json
├── dist/                         # Compiled output (generated by build)
├── package.json
└── image.png                     # Extension icon
```

## Registered Types

### Structs

**`channel_config`**
| Field | Type |
|-------|------|
| `active` | boolean |
| `value1` | float |
| `value2` | float |

**`device_data`**
| Field | Type |
|-------|------|
| `name` | string |
| `oldName` | string |
| `createdOn` | integer |
| `createdBy` | string |
| `modifiedOn` | integer |
| `modifiedBy` | string |
| `channel1` … `channel5` | struct:eez-ext-sample/channel_config |

### Enum — `ChannelStatus`

| Member | Value |
|--------|-------|
| `CHANNEL_ERROR` | -1 |
| `CHANNEL_IDLE` | 0 |
| `CHANNEL_ACTIVE` | 1 |
| `CHANNEL_DONE` | 2 |

### Action Components

**`GetData`** (`flowComponentId: 10001`, native: `Sample_GetData`)
- Property: `result` — assignable expression of type `device_data`

**`SetData`** (`flowComponentId: 10002`, native: `Sample_SetData`)
- Property: `data` — expression of type `device_data`
- Logs the data value to the EEZ console when executed in simulation

### Object Variable Type — `Data`

A configurable variable type with a built-in dialog editor. Supports:
- Named/unnamed status display (green when named, gray when unnamed)
- 5 channels, each with `active`, `value1`, and `value2` fields
- Creation and modification metadata tracking

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/)
- [EEZ Studio](https://www.envox.eu/eez-studio/)

### Install dependencies

```bash
npm install
```

### Build

```bash
npm run build
```


The compiled output is placed in `dist/index.js`.

### Install in EEZ Studio

In EEZ Studio, open **Settings > Extensions** and install the extension as a Project Editor Extension (PEXT) from the local folder. Don't use network disks.

## Customization Guide

To adapt this sample to your own use case:

1. **Rename the extension** — update `name` and `id` in `package.json`.
2. **Add or modify action components** — edit `src/flow-components.ts` and add entries to the `ACTIONS` array.
3. **Change struct fields** — update `registerStructureVariableType` calls in `src/index.ts`.
4. **Change enum values** — update `registerEnumVariableType` in `src/index.ts`.
5. **Customize the object variable dialog** — edit `src/object-variable-types.ts`.
6. **Implement native execution** — replace the `nativeExecute` string with your C/WASM function name, or implement the `execute` function directly in TypeScript for simulation.
